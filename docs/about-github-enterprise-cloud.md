GitHub Enterprise Cloud هو خيار نشر لمنصة GitHub Enterprise، ويوفّر مزايا متقدمة على GitHub.com منها:

* مصادقة SAML
* دقائق إضافية لخدمة GitHub Actions
* تقييد إشعارات البريد الإلكتروني على النطاقات الموثوقة فقط
* إمكانية نشر مواقع GitHub Pages بشكل خاص
* إدارة المستخدمين المُدارين
* قواعد خاصة للمستودعات
* تقارير امتثال من GitHub

للاطلاع على قائمة المزايا الكاملة المتوفرة في GitHub Enterprise Cloud، راجع [صفحة الأسعار](https://github.com/pricing).

## ما الفرق بين GitHub Enterprise Cloud والخطط الأخرى؟

الفرق الأساسي بين GitHub Enterprise Cloud والخطط الأخرى على GitHub.com هو الوصول إلى حساب المؤسسة، والذي يمنح المدراء مركزًا موحّدًا لإدارة السياسات والفوترة والهوية والوصول والتدقيق لعدة منظمات.

لمزيد من التفاصيل حول GitHub Enterprise Server وكيف يختلف عن GitHub Enterprise Cloud، راجع "[حول GitHub Enterprise Server](/admin/overview/about-github-enterprise-server)".

## حول أنواع المؤسسات

هناك عدة أنواع من المؤسسات يمكن الإطلاع عليها في الوثائق الرسمية لـ GitHub.

## حول موقع البيانات

إذا اخترت مؤسسة مع مستخدمين مُدارين، يمكنك أيضًا اختيار مكان تخزين بيانات شركتك.

افتراضيًا يتم تخزين البيانات على GitHub.com في الولايات المتحدة الأمريكية. لمزيد من التفاصيل راجع الوثائق المتعلقة بسياسة حفظ البيانات.
